深入学习Java8 Stream

发表于 2019-03-10 | 分类于 Java |

What

Stream是Java8中新引入的一个特性，它旨在帮助程序员站在更高的抽象层次上对集合进行操作。
Stream以管道(Pipeline)的形式支持对集合的操作，每次操作都会产生个新的Stream。

Stream Pipeline可以分为三个部分：Stream Source、一个或者多个中间操作(Intermediate Operation)以及一个终止操作(Terminal Operation)。

Stream Source可以是Collections、Arrays、Generators Function或者是其他任何提供了访问其元素的数据源。

每一个Intermediate Operation都会生成一个新的Stream。Intermediate Operation又可以分为两种类型：Stateless Operation(无状态操作)以及Stateful Operation(有状态操作)。
Stateless Operation是指操作是无状态的，不需要知道集合中其他元素的状态，每个元素之间是相互独立的，比如map()、filter()等操作。Stateful Operation是指有状态操作，操作是需要知道集合中其他元素的状态才能进行的，比如sort()、distinct()。

Terminal Operation从逻辑上可以分为两种：短路操作(short-circuiting)以及非短路操作(non-short-circuiting)。短路操作是指不需要处理完所有元素即可结束整个过程，非短路操作是需要处理完所有元素之后才能结束整个过程。

Terminal Operation从行为上说可以分为3类：聚合操作(aggregations)、搜索操作(find、match)以及有副作用的foreach(forEachOrdered)操作。

spliterator()方法会返回一个Spliterator对象。Spliterator是个接口，见名思意，它主要提供两个能力：

  • split：对应能够并行的场景，Spliterator可以划分成多个Spliterator对象，每个Spliterator对象处理部分数据。
  • iterator：访问数据源中的每一个数据。

继续跟踪进去查看它的定义(只列出主要方法)：

从它的注释中可以了解到：Spliterator对象会遍历和分割它所关联的数据源，也就是前面所说的 Stream Source。
Spliterator对象提供了两个访问数据源元素的方法:

  • boolean tryAdvance(Consumer<? super T> action);:这个方法每次会处理数据源中的一个元素。如果元素已经处理完成，tryAdvance()返回false，否则的话它会把当前游标下的元素传递给action处理，并把游标加一，然后返回true。
  • void forEachRemaining(Consumer<? super T> action);:从实现中可以看到，该方法会一次性处理所有剩余的元素。

前面介绍过，Terminal Operation可以分为短路操作和非短路操作。对于短路操作，每次处理一个元素，需要调用tryAdvance()方法，而对于非短路操作，可以一次性批量处理多个数据，调用forEachRemaining()方法。

除此之外，Spliterator还可以对数据源进行分割，这样多个线程就可以同时处理数据。如果Spliterator可以被分割的话，Spliterator<T> trySplit()方法会返回一个新的Spliterator对象，新的Spliterator对象会切片部分原始数据，否则的话该方法返回null表明不能分割，数据需要被串行处理。

该方法中会构造一个ReferencePipeline.Head对象，ReferencePipeline.Head的继承图如下所示：

Head继承自ReferencePipeline，ReferencePipeline是Stream Pipeline中Pipeline源(Pipeline source)和Pipeline中间阶段(intermediate pipeline stage)的公共父类。ReferencePipeline类实现了Stream接口，所以每一个ReferencePipeline子类生成的对象都属于Stream对象。而ReferencePipeline类又继承自抽象父类AbstractPipeline，该类中字段定义如下图:

  • sourceStage:指向pipeline chain中第一个stage。如果当前stage就是source stage，则该字段指向自身。
  • previousStage:指向当前stage的上一个stage，如果当前stage是sourceStage，则该字段为null。
  • nextStage:当前stage的下一个stage，如果当前stage是最后一个stage，则该字段为null。
  • sourceSpliterator:源spliteraror，只有当前stage为head的时候才有效。
  • linkedOrConsumed:当前stage是否有下游stage或者是否被消费，默认为false。
  • depth:如果是串行的话则表示当前stage到source stage直减中间stage的个数，如果是并行的话则表示当前stage到上一个stage的个数。
  • sourceAnyStateful:如果整个pipeline chain中有任何一个有状态操作则为true，反之则为false。该字段只在source stage中生效。
  • parallel:当前计算是否能够并行。

Sink

Sink类继承自Consumer，是属于Consumer类的扩展。

在第一次调用accept()接收新数据前需要首先调用begin()方法通知Sink即将有数据到来(如果能够附带数据量大小则更佳)。在接收完所有的数据之后需要调用end()方法表示数据已处理完。

在调用end()之后，如果想要再次调用accept()接收数据，则必须再次首先调用begin()方法。

通过cancellationRequested()方法，Sink可以通知调用方自己暂时不想再接收数据。

Sink有两种状态：初始状态(initial state)和激活状态(active state)。

Sink起始于初始状态，调用begin()会使得状态转换至激活状态，accpet()方法只有在激活状态下才有效。调用end()方法会让Sink的状态重新回到初始状态。
